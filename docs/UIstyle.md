# 🌙 AI 对话智能体前端设计文档

（参考 fastapi-radar 深色科技风）

---

## 1. 项目概述

**产品类型**：类似 ChatGPT 的 AI 对话智能体前端
**定位**：提供一个高可用、可扩展的对话界面，支持多会话、多智能体配置、上下文管理和工具调用展示。

**设计关键词**：

* 深色 / 科技感 / 冷色系
* 中心是“对话区”
* 侧边管理：会话、智能体配置、知识/工具
* 卡片式布局，信息有层次但不过度花哨

---

## 2. 核心使用场景

1. **普通用户对话**

   * 像 ChatGPT 一样发消息、读回复。
   * 支持连续对话，保留上下文。

2. **开发者 / 高级用户**

   * 创建、编辑不同的智能体（System Prompt、参数、工具开关）。
   * 管理会话、查看模型调用详情、调试提示词。

3. **运营 / 产品**

   * 使用固定配置的智能体处理特定任务（客服、写作、总结等）。
   * 可能需要从预设模板创建对话。

设计时要保证：**新手能马上用，高级用户有地方“折腾”。**

---

## 3. 信息架构（IA）

### 3.1 顶级结构

建议整体有 3–4 个一级模块：

1. **Chats / 对话**

   * 主工作区，默认打开。
   * 左侧：会话列表。
   * 中间：当前对话。
   * 右侧（可折叠）：智能体配置 / 上下文信息。

2. **Agents / 智能体配置**

   * 配置不同人格/用途的 Agent（类似“角色卡”）。
   * 每个 Agent 包含名称、头像/图标、系统提示、模型参数等。

3. **Knowledge / Tools（可选模块）**

   * 绑定知识库、文档、插件、API 工具等。
   * 控制哪些 Agent 可以使用哪些资源。

4. **Settings / 设置**

   * 主题、语言、快捷键、隐私相关开关等。

### 3.2 主对话页结构

主对话页（Chats）内部结构建议：

* **左侧：会话列表 Sidebar**
* **中间：对话区**
* **右侧：当前智能体 / 会话设置（可选 & 可折叠）**

---

## 4. 全局布局与视觉风格

### 4.1 布局模型

仿 fastapi-radar 的感觉：

* **左侧垂直导航（窄宽度）**：模块切换（Chats / Agents / Settings...）
* **中间主区域**：随模块变化

  * 在 Chats 模块下，再次拆为：

    * 左：会话列表
    * 中：对话
    * 右：配置抽屉（可隐藏）

这样层级非常清晰，业务扩展也方便。

### 4.2 深色主题配色（示意）

> 下面是“设计思想”，不是硬编码颜色，你可以按自己偏好替换。

* 页面背景（最底层）：接近黑的深蓝灰，如 `#020617 ~ #020818`
* 内容背景（卡片）：稍亮一点的深灰，如 `#0f172a ~ #111827`
* 内部块背景：`#1e293b` 左右
* 边框：`#1f2937 ~ #273549`
* 主色 Accent：青蓝/青绿（例如 `#0ea5e9` 或 `#22c55e`），用于：

  * 当前选中菜单项
  * 高亮按钮
  * 聊天中的“我方气泡”或光标

对比度：深背景 + 浅文字（白色或浅灰），再辅以弱对比的说明文字。

### 4.3 字体与排版

* 字体：Inter / SF Pro / system UI sans-serif 系。
* 字号：

  * 页面标题：20–24px
  * 区块标题：16–18px
  * 对话文本：14–16px
  * 辅助信息：12–13px

文本风格：

* 用户消息、AI 消息都保持易读（足够行高和行宽）。
* 系统提示/标签/时间戳使用更小号、浅色。

---

## 5. 核心页面设计

### 5.1 对话页（Chats）

#### 5.1.1 左侧会话列表

用途：管理多个会话，切换上下文。

元素：

* 顶部：

  * 当前智能体下拉 / Tabs（支持配置不同 Agent）
  * “新建会话”按钮
* 列表项：

  * 会话标题（自动以首轮对话命名，可编辑）
  * 更新时间 / 简短摘要
  * 选中态高亮（背景稍亮 + 左侧主色条）
* 操作：

  * 删除、重命名、置顶
  * 搜索会话（顶部搜索框）

交互风格：
**不需要太花时间在视觉特效上，关键是稳定可用、快速切换。**

---

#### 5.1.2 中间对话区

这是整个产品的「舞台」。建议结构：

* 顶部栏（可选）：

  * 当前会话标题
  * 关联 Agent 名称 + 图标
  * 一些辅助按钮：清空对话、导出、重置上下文

* 消息区域：

  * 时间顺序消息流
  * 用户消息、AI 消息有明显区分：

    * 左右对齐（User 右侧，AI 左侧）或
    * 同侧，但颜色和头像不同（更适合桌面宽屏和 markdown 内容）

* 输入区：

  * 多行输入框，支持自动增高
  * 右侧：发送按钮
  * 左侧/上方：可放

    * 模式切换（例如：问答 / 写作 / 翻译）
    * 文件上传按钮
    * 快捷插入模板按钮

##### 消息卡片样式

每条消息可以看作一个「卡片」：

* 背景：略微高于页面背景的颜色
* 内部结构：

  * 顶部小行：头像 + 名称（User / Agent 名）+ 时间
  * 内容：富文本/Markdown/代码块
  * 底部操作（可选）：复制、重发、停用、反馈（👍/👎）

AI 消息生成过程中：

* 显示“正在输入...”动画（光标闪烁、小点点动效）
* 支持中途停止生成按钮（Stop）

---

#### 5.1.3 右侧配置 / 信息栏（可折叠）

用于当前会话的侧信息：

* 当前使用的 **Agent 配置**：

  * 角色名称
  * 简短描述
  * 图标/头像
* 本会话特定设置：

  * 模型 / 温度 / 最大 Token 等（如果你想暴露给用户）
* 上下文信息：

  * 导入的文档列表
  * 绑定的知识库/插件
* 高级信息（可选）：

  * 系统 Prompt 预览（只读/可编辑）
  * 一些调试信息，如 Token 统计（偏开发者向）

该栏可以使用抽屉式（Drawer）或固定卡片，**初次使用时建议默认展示，用户可以收起。**

---

### 5.2 智能体配置页（Agents）

让用户像“捏人”一样配置不同的智能体，方便复用与分享。

元素：

* 智能体卡片列表：

  * 名称、头像、标签（用途：翻译/助教/代码助手…）
* 详情配置：

  * 基本信息：名称、描述、图标颜色
  * System Prompt（多行文本/Markdown）
  * 模型参数：模型名、温度、Top-p 等
  * 可用工具 / 插件开关（如果有）
* 操作：

  * 新建智能体（可从模板）
  * 复制已有智能体
  * 导出 / 导入配置（JSON 或自定义格式）

视觉上仍保持卡片式、一致深色风格，不出现监控类东西。

---

### 5.3 设置页（Settings）

统一放产品级偏好设置：

* 主题：浅色 / 深色 / 跟随系统
* 语言设置
* 快捷键（发送：Enter / Ctrl+Enter）
* 隐私：是否在本地保存对话、缓存期等等
* 关于 / 版本信息

---

## 6. 组件与交互细节（面向 AI 产品的特化）

### 6.1 Markdown / 代码块

因为是 AI 对话，强烈建议对以下内容有专门样式：

* 标题、列表、引用块
* 代码块（Monospace 字体，带复制按钮）
* 表格（有边框 + 滚动）

这部分可以延续深色科技风，代码块背景稍暗、边框淡淡高亮。

---

### 6.2 流式生成（Streaming）

* 当 AI 正在生成内容：

  * 该条消息卡片已出现，内部内容从上到下逐行出现
  * 显示一个“停止生成”按钮
* 生成结束：

  * 停止动效
  * 可以开放“重新生成” 或 “修改后重试”。

---

### 6.3 错误与状态提示

* 请求失败 / 超时：

  * 直接在对话流中插入“系统消息卡片”说明出错原因
  * 不要仅弹 Toast，避免信息丢失
* 限制类提示（额度、频率）：

  * 使用醒目 Badge / 卡片说明，但尽量不打断流程。

---

### 6.4 快捷操作

* 在消息 hover 时：

  * 显示“复制”、“重新提问”、“设为系统提示”、“加入知识库”等按钮。
* 在输入框上方可设置：

  * 历史问题建议
  * Prompt 模板快速插入

这些都是可选，但强扩展性设计有利于你逐步迭代。

---

## 7. 响应式设计（针对 Chat UI）

* 宽屏（桌面）：

  * 左：会话列表
  * 中：对话
  * 右：配置栏
* 中等屏宽：

  * 可隐藏右侧配置栏，点击图标切换
* 手机：

  * 顶部或底部使用 Tab 切换“会话列表 / 对话”
  * 配置栏改为单独的抽屉页面

核心原则：**对话区永远是第一优先，其他区域让路。**

---

## 8. 动效与细节风格

* Hover：卡片、按钮略微提亮 + 轻微阴影
* 点击：缩放 1–2% 或加深色块即可
* 模态和抽屉：轻微位移 + 透明度动画
* 非必要不要用大范围动态背景，避免干扰阅读对话内容

---

## 9. 可扩展性：为未来功能预留空间

设计时可以提前预想：

* 多模态（图片、音频）消息：

  * 每条消息卡片内部支持图片缩略图、音频播放条。
* 工具调用结果展示：

  * 在 AI 消息中嵌入「结构化卡片」，如表格、步骤、日程卡片等。
* 协同编辑 / 多人视角：

  * 会话列表项可以展示“共享”标记。

在文案和布局上，尽量**不要把界面写死为“聊天机器人”**，而要用更通用的词：
如“智能体”“对话”“会话”“配置”等，这样你将来扩展到 agent 平台也不会违和。
